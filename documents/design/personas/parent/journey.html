Gaps & open questions

BACKLOG GAP

S12 — Notification settings: No FHS ticket exists for granular notification preferences (per-event-type push/email toggles on mobile). Needs a Sprint 3 or 5 ticket under the FHS-47 epic or a new one.

OPEN Q

Slug uniqueness UX (S5): what happens if the user's preferred slug is taken mid-onboarding? Design shows suggestions — but the suggestion algorithm (prefix / suffix / number?) is not yet specced. Needs a decision before FHS-29 implementation.

OPEN Q

Upgrade flow redirect (S13 → S14): Does Stripe Checkout open in a new tab or replace the current route? ADR 0004 defers to Stripe Checkout (hosted page), which means a tab-replace. Confirm UX impact — a new tab risks the parent losing their dashboard state.

OPEN Q

Mobile app screens: Screens 1–11 and 13–15 are web (laptop). Only S12 is mobile. No native iOS/Android sprint is currently scheduled in the Sprint 0–6 plan — all surfaces are mobile-web responsive. Flag if a native app changes the notification settings interaction model significantly.

DESIGN NOTE

Error branches: S5 shows a dashed error hint for slug collision. S4 shows a wrong-email recovery note. Full error-state screens (form validation shake, server error) are annotated inline rather than as separate mockup frames to stay within token budget. Expand in the component-level spec.
