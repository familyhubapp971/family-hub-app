Parent Persona · Feature Design

Assignments + Family Admin

Assign chores and homework to any family member, review photo proof, and manage the full family roster — all from one parent-facing control panel.

👩‍💼

Primary Persona

The Coordinating Parent

  • Goal Keep every child on track without being their alarm clock
  • Pain Tasks forgotten, responsibilities unclear, no visibility
  • Context Manages 2–4 children aged 6–16, uses web on desktop
  • Role Admin-level access: creates tasks, invites members, sets roles

Legacy Reference — family-hub v1

Assignments / Tasks

  • Assignments.tsx — per-subject cards (Math/English/Science/Art/General), toggle completion, inline edit, file attachments, Claude Vision AI extraction from uploaded images/PDFs
  • MyTasks.tsx — unified task feed pulling from assignments, announcements, and standalone tasks; priority inferred from due-date proximity; kid-facing "today" view
  • No recurring tasks; no photo-proof gating; single member-per-assignment only

Admin Panel / Roles

  • AdminPanel.tsx — 5 tabs: Balance, Savings, History, Users, App Info; Users tab has per-member permission toggles (16 flags), inline role edit, add-user form
  • ProfileSelector.tsx — emoji + magical-phrase auth; no JWT; Child members excluded from login
  • Single-family, no tenant isolation; roles stored as plain text; no email/magic-link invites

What's Built vs Backlog Gap

✅

Already Shipped

  • ✓

    Tenants table — FHS-2

    name, slug, timezone, currency columns live; RLS enforced per tenant_id

  • ✓

    Members table with role enum — FHS-3

    role: admin | adult | teen | child | guest; tenant_id FK; RLS policies

  • ✓

    Activity logs — FHS-4

    Captures admin actions; usable as audit trail for role changes + task approvals

  • ✓

    Supabase Auth providers — FHS-187, FHS-189, FHS-191

    Email+password, OAuth (Google), magic-link; JWKS / ES256 middleware; users mirror

  • ✓

    Supabase Storage provisioned — FHS-189

    Private bucket available; tenant-prefixed paths can be wired for photo proofs

🚧

Backlog Gap

  • !

    tasks + task_proofs tables

    No schema, no migration, no RLS. Must include tenant_id, assignee_member_ids (array FK), recurrence, photo_proof_required, reward_count.

  • !

    member_invites table + magic-link flow

    No invite model; invite token generation, email send, redemption endpoint all missing.

  • !

    Tasks CRUD endpoints

    POST /api/tasks, GET /api/tasks, PATCH /api/tasks/:id, DELETE; GET /api/tasks/proof and proof approve/reject.

  • !

    Role-based authorization guards at API edge

    members.role checked in middleware; admin-only routes guarded; child members cannot approve or invite.

  • !

    All UI screens

    My Tasks (kid), AssignTask modal, Parent Overview table, Photo Proof Review, Member List, Edit Member, Invite, Family Settings — all net-new.

  • !

    Sticker-balance side effects

    Proof approval must atomically update members.sticker_balance + log activity. Needs careful transaction or event pattern.
