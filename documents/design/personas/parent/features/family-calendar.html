Legacy Reference

What the v1 (family-hub) app shipped — neutral, informational only

  • 1.Two separate tabs — "School" and "Home" in a CalendarSection switcher; no combined view. Each rendered its own weekly column list.
  • 2.Home categories: Family / Health / Fun / Chores / Appointment with emoji + pastel colour tags; activities sorted by 12-h time string.
  • 3.School model: per-day start/end time + activity rows with Morning/Afternoon tag, room, and uniform type (Normal / Sports CPE); no multi-child support.
  • 4.No recurrence, no conflict detection, no iCal import, no push reminders — all day-level manual entry only.
  • 5.Single-family, no tenant_id — data model used flat home_activities and school_activities tables with no RLS.
  • 6.SaaS gap: the new architecture needs member_ids[], tenant_id RLS, timezone-aware timestamps, recurrence_rule (RRULE), and external prayer-time syncing — none of which existed.

✓ Member profiles (family members, member_id)
✓ Tenant timezone field (Asia/Dubai default) — FHS-2
✓ Weeks / family-time backbone — FHS-3
✓ Supabase Auth + JWT middleware (FHS-178 epic)
✓ Design system: Card, Button, Badge, Input, neo-brutalist tokens
