Legacy Reference (family-hub monolith — neutral, for context only)

  • Announcements.tsx — task-style list with done/pending toggle, category badges (School/Event/Health/Trip/Reminder/General), deadline countdown, and inline edit. Scoped to single-tenant, no real-time feed.
  • NoticeboardSection.tsx — tab switcher combining Assignments + Announcements under one "Noticeboard" heading. Simple 2-tab UI, no pinning or audience targeting.
  • announcements.md — ACs cover list/create/toggle/delete/attach + AI extraction from uploaded images. No read receipts, reactions, or push notifications defined.
  • Legacy model: announcements + announcement_attachments only. Multi-tenant tables, reactions, reads, and audience filter are absent.
  • AI extraction flow (Claude Vision API) is the standout legacy innovation — worth preserving in SaaS rebuild.
  • No push notification infrastructure existed; email/WhatsApp notifications were best-effort on assignment only.

Architecture Mapping

5 key actions mapped across USER → WEB → API → DATA → EXTERNAL

User

Web (React)

API (Hono)

Data (Postgres)

External

Sarah fills form, taps "Post Announcement"
AnnouncementForm
validates Zod schema locally, calls POST /api/announcements
Validates body (Zod), injects tenant_id from JWT, inserts row + enqueues push job
INSERT INTO announcements (tenant_id, author_member_id, title, body, audience_filter, pinned, expires_at)
—
Members receive push on mobile devices
Push notification opens app → navigates to Noticeboard
Worker job reads push_tokens for audience members, calls Expo/FCM
Read member_push_tokens (tenant_id, member_id, token, platform)
Expo Push API / Firebase FCM → iOS & Android lock screen
Member opens / scrolls past announcement
AnnouncementCard fires POST /api/announcements/:id/reads on viewport enter
Extracts member_id from JWT; idempotent upsert
UPSERT INTO announcement_reads (tenant_id, announcement_id, member_id, read_at) ON CONFLICT DO NOTHING
—
Member taps ❤️ on announcement
ReactionPicker → POST /api/announcements/:id/reactions {emoji: "❤️"}
Upsert: one reaction per member per emoji per announcement; toggle off if same
INSERT INTO announcement_reactions (tenant_id, announcement_id, member_id, emoji) ON CONFLICT DELETE (toggle)
—
Author taps "Nudge 👋" on unread member in dashboard
ReadReceiptDashboard → POST /api/announcements/:id/nudge {member_id}
Rate-limit: 1 nudge per member per announcement per 24h; enqueues targeted push + email
INSERT INTO nudge_log (tenant_id, announcement_id, member_id, sent_at)
Expo push + Supabase email (targeted, not broadcast)
