Coordinating Parent — Critical Features

How the everyday paying user does the things they came for, under the new SaaS architecture. Each card opens a deep-dive page with screen mockups, an architecture map (USER / WEB / API / DATA / EXTERNAL), and a "shipped vs gap" footer that cross-references FHS tickets.

How to read these

  • Legacy reference — short summary of how the legacy family-hub (single-tenant) version did it. We're not copying — just learning.
  • Screen mockups — what the actual UI looks like under the new SaaS architecture. Real copy, real device frames, real component idiom from @familyhub/ui.
  • Architecture map — swimlane (USER / WEB / API / DATA / EXTERNAL) showing which layer does what. Mirrors the format used in whats-shipped.html.
  • Shipped vs gap — the schema/infra that already exists in this repo (FHS-2, FHS-3, FHS-4, etc.) vs what still needs ticketing. Tells you "how much further until this feature can ship".
