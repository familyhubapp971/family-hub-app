Gaps and open questions — Persona 3 retention depends on these
Unticketed screens (BACKLOG GAP)
S2 — Public roadmap page
This is the primary hook for Persona 3. Without it, he has no signal the team ships. No FHS ticket exists. Needs a dedicated Jira epic covering roadmap CMS, vote system, comment thread, and public visibility permissions.
S3 — Feature detail / comment page
Public-facing feature spec page with vote count, comments, and sprint target. Tied to S2 ticket. No FHS ticket.
S7 — API tokens and developer settings
Public API v1 is the top-voted roadmap item (312 votes). The settings surface to manage tokens and webhooks must exist before launch. No FHS ticket. Suggest child under FHS-205 tech-debt or a new Sprint 4 epic.
S8 — Referral program
Persona 3's outsized advocacy value is only captured if there is a structured referral loop. No FHS ticket. Needs Stripe billing hook (FHS/ADR-0004), referral tracking, and credit application logic.
S9 — Changelog / "What's new"
Changelog is the retention mechanism between releases. Tech-fluent users check it. No FHS ticket. Even a simple markdown-driven page on familyhub.app/changelog reduces churn signals.
S10 — Public share / screenshot generator
This persona posts about products. If sharing is frictionless with a branded screenshot, each post is a free acquisition event. No FHS ticket. Lower priority than S2/S7 but high ROI per build hour.
Open questions
Who owns the public roadmap content?
Is this a CMS-backed page (Notion embed? Custom admin?), a static markdown file, or a third-party tool like Canny? Decision needed before Sprint 2.
Public API v1 — scope before launch?
312 votes is the highest count on the roadmap. Does a read-only API (calendar events, task status) ship before GA? Or post-launch? Needs a dedicated ADR and FHS ticket.
Referral credit — Stripe mechanism?
One month free = a Stripe coupon applied at renewal, or account credit? Needs alignment with ADR-0004 billing model before implementing S8.
Beta features gate — which tier?
Should beta features only show in S7 for Family Pro users, or all trial users? Aligns with FHS billing tiers but needs explicit product decision.
Screenshot generator — privacy model?
S10 generates a branded dashboard image. Must confirm no real child names or identifiable data appear in the shareable asset. Requires a server-side sanitised render, not a client screenshot.
Changelog — subscription mechanism?
"Subscribe to updates" on S9 implies email notifications for releases. Needs Supabase transactional email or a third-party (Resend, Loops). Scope?
